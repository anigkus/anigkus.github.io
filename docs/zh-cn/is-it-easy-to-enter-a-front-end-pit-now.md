> <br/>&nbsp;&nbsp;&nbsp;&nbsp; 记得刚学编程那会,基本前端也会学,但是刚开始学时,老师不会告诉你有啥前端开发工具(Dreamweaver、Frontpage),直接纯手工.比如记事本写,那时多么痛苦,根本没有任何提示信息,刚开始基本是靠完全抄书或者脑子记(那时头发掉的特快,霸王完全不够用啊,差点就地中海或者光头强了),慢慢的手工敲打的熟练了,老师也会告诉你使用一些工具,这样可以提高效率.比如Notepad、Editplus、TextEdit,后来才有什么Sublime、Vs Code,自从会使用工具之后,头发浓密多了,在也不想使用记事本靠脑子死记硬背了,那断不堪回首的时光,真是惨不忍睹啊,但是会使用多种前端编程工具只是入门,核心还是要理解各个框架的核心原理和架构模型.这样就算怎么变,入门和理解起来都很快.你想现在入门个前端那时不容易啊,前端的技术进几年发展变化太快了,完全跟不上节奏,基本每个月都会出来一个新框架或者新开发工具,现在就让我们来看看现在入坑前端需要懂多少东西吧,这在以前着分分钟都是大牛啊,现在学会这些还只是个菜鸟,哎,不容啊!<br/>

# 切图工具

&nbsp;&nbsp;&nbsp;&nbsp; 以前前端和美工是一个技术工种啊,随着互联网的发展,美工叫视觉设计师或者UI设计师,前端已经是另一个就是单独的工种了,但是UI工程师把页面设计出来,还是需要前端负责把一张整体图片,切成一个个小小的图标或者图片按钮,然后使用html+css组装起来,那么现在的前期切页面或多或少都会用到以下的工具.

* HTML+CSS: 浏览器底层支持的语言,必须用到.

* Adobe PhotoShop(PS): 编辑图片或图片处理.

* Adobe Illustrator（AI): 矢量图形处理工具.

* Adobe Dreamweaver(DW): 网页代码编辑器.

* Cutterman: PhotoShop中的插件,能够自动将你需要的图层进行输出.

* LanhuApp: 蓝湖是一款高效的产品设计协作平台.

* Avocode: 主要功能包括设计、导出素材、获取图像规格等.


# 编程知识

&nbsp;&nbsp;&nbsp;&nbsp; 以前学前端,基本Html+Css+javaScript三招可以走遍天下,现在根本不敢出门啊,哈哈,因为现在前端的编程语言,每一个3-5门,不敢说是前端程序员啊,你看看下面这些语言,前端都会写,只是某些使用多点,某些使用少点而已.

* HTML+JavaScript: 这2个语言组合是现在主要浏览器的大多数元素.

* PHP、Python、Ruby、BashShell: 会用到这些语言的基本是是有点道行的,不是说语言难,而是前端能用到这些说明基本往全栈开发去发展来,厉害大神.

* NodeJs: 服务端的 JavaScript,可以把js在服务端运行,也不一定需要在客户端的浏览器中运行的语言.

* ActionScript: Adobe公司完全的面向对象的编程语言,主要用来编写动画,只能在Flash播放器中的虚拟机里面运行.

# 服务部署

&nbsp;&nbsp;&nbsp;&nbsp; 如果项目完成了,需要部署到公有的网络去测试,看看效果,你也有可以在本机部署Apache\Tomcat\Jetty,但是如果你想给别人用,但是自己又不想买域名或者云服务器,可以使用以下Paas产品去测试下.

* GAE(Google App Engine) : 是一种让您可以在 Google 的基础架构上运行您的网络应用程序,可以直接运行Web程序,是一种Paas云平台模式,稳定,但是国内访问不了.

* SAE(Sina App Engine): 新浪研发中心推出的一个公有云计算平台,可以直接运行Web程序,是一种Paas云平台模式,不稳定,可以支持二级域名.

* BAE(Baidu App Engine): 是百度智能云提供的PaaS平台.

* OpenShift: 是红帽 云计算平台使开发人员能够创建、测试和运行他们的应用程序,并且可以把它们部署到云中,也是Paas云平台模式,这个在社区说反应比较不错.


# 技术社区

* MDN : Mozilla开放 Web 技术的开发者而打造的社区.

* MSDN: 微软的技术文档中心.

* StackOverflow: 问答平台.

* Github:开源界最流行=代码托管服务平台.

* Google Developer/web: google开发者社区.

* NodeParty: 一个面向 Node.js 爱好者的社区组织.

* D2前端技术论坛.

* QCon是由InfoQ主办的综合性技术盛会.

* JSConf China(JS中国开发者大会).

* YUI开发者大会.

* 腾讯alloyteam Web团队.

* W3cplus是前端行业技术博客.


# 移动终端

&nbsp;&nbsp;&nbsp;&nbsp; 以前PC是主流用户的使用的平台,好多公司都没有Wap端,如果需要Wap端,也基本是判断浏览器头,然后利用window.onresize函数来做伸缩控制,没有统一移动端解决框架,自从智能手机普及之后,手机端和H5端已经成为主流,这时移动端的框架就如雨后春笋般涌现出来,来看看现在如果开发移动端应用需要哪些技术栈吧

* jQuery Mobile : jQuery移动端解决方案.

* HTML5: HTML5是互联网的下一代标准,对视频、音频、图像、动画以及与设备的交互都进行了规范.

* CSS3: 当今html5中主流样式规范标准.

* Iphone/Ipad/IOS: 使用Object-C或者Swift开发语言.

* Android: 使用Java或者Kotlin.

* Angular/React.js/Vue.js: 构建Web应用程序的顶级JavaScript几个框架.

* Flutter/Cordova/Weex/Xamarin:  几个跨平台的语言解决方案,一套代码,多平台运行,Android/Ios/Web都行,Flutter比较有未来趋势,社区比较看好这个.


# 调试工具

&nbsp;&nbsp;&nbsp;&nbsp; 当写完某段功能代码,不知道运行的效果或者有没有问题,最好的方式当时是运行并调试下,看看逻辑是否预期,那么前端开发其实调试工具还是蛮多的,因为现在的浏览器都待用诊断工具,很方便排查问题,看看现在都有哪些前端诊断工具

* FireBug: Firebug是firefox下的一个扩展,能够调试所有网站语言.

* Web Inspector: Chrome或者Safari浏览器都带有相关的开发者工具.

* YSlow/Smushit: YSlow是Yahoo发布的一款基于FireFox的插件/smushit是一款免费在线图片压缩小工具.

* IEDeveloperToolBar/IETester: 在IE浏览器的诊断工具/以及测试IE不同版本兼容性的工具),IE基本快淘汰了,最新的微软浏览器是Microsoft Edge(听说也快挂了.

* SuperPreview/JsBeautifier: SuperPreview 浏览器兼容性测试/Jsbeautifier是一款JS代码美化工具.

* Fiddle/WireShark/Tcpdump: 三款都是抓包工具,前面2跨可以直接操作系统应用程序,Tcpdump是Linux一个命令,抓4层协议(包括4层)以上的包结果.


# 前端安全

&nbsp;&nbsp;&nbsp;&nbsp; 总体来说前端安全分两方面,一个是输入一个是输出,就是如何防止正常用户逻辑来破坏系统,但是如果不是正经人那么前端的安全基本就是无效,因为他们可以直接请求后端接口,直接绕过浏览器的判断,直接进入后端,现在来看看前端有哪些安全是可以控制的呢?

* SQL注入/文件上传漏洞: 某些SQL关键字不能提交或者替换掉/文件后缀或者白名单等,防止上传病毒脚本文件到服务器.

* XSS/CSRF: XSS 全称跨站脚本攻击(输入过滤、输出编码、安全编码、HttpOnly Cookie等)/CSRF跨站请求伪造(Referer Check、HttpOnly).

* FBML/caja/sandbox/ADsafe: 都是前端沙箱解决方案,就是解决如何安全地嵌入第三方js.

# 开发工具

&nbsp;&nbsp;&nbsp;&nbsp; 前端开发工具基本就前面说的那些了,但是现在很少用纯文本的编辑器去写前端代码了,因为没有提示还是很痛苦(大牛除外了),但是用的比较多的就下面几个.

* Visual Studio Code: 微软开源针对于编写现代 Web 和云应用的跨平台源代码编辑器.

* Sublime: SublimeText是一款开源跨平台代码编辑器（CodeEditor）软件.

* Webstorm: WebStorm 是jetbrains公司旗下一款JavaScript 开发工具,收费的.

* Dreamweaver: Adobe旗下的集网页制作和管理网站于一身的所见即所得网页代码编辑器.


# 软性技能
&nbsp;&nbsp;&nbsp;&nbsp; 如果是编码是我们的硬性技能,那么也需要提高我们自身的软性技能,因为东西都是两面性,需要软硬兼施,才能做到恰到好处,那么需要培养我们哪些软性技能了,软技能涉及自我管理、人际沟通、职业发展、领导管理各个方面,解决问题的能力,技术总结分享,跨团队沟通或开发能力,对需求的理解能力以及项目管理能力,还有框架模块化思想及分支版本管理,也没交互设计及服务可用性知识等,这些都是非标品,没法和技术相比,能用一个特定的值来衡量,只能因事而论.


# 知识储备
&nbsp;&nbsp;&nbsp;&nbsp; 知识储备不仅仅是前端经常使用的某个框架或者JS库就可以了,你还需要了解操作系统原理、计算机网络模型、常用的几种算法、软件工程模型、软件测试流程、数据结构、面向对象编程、切面编程、MVC/MVVM/SPI架构模式、函数式编程、JS设计模式等,这些都要去补充相关知识面,这个其实就是技术的广度了.

# 构建工具 
&nbsp;&nbsp;&nbsp;&nbsp; 构建工具只是帮助我们更快、更好、更有效率提高代码的整体质量,降低开发复杂度,对项目整体管理,解决项目中多依赖的问题,实现前端某些流程能够自动化,释放一些重复性的工作,让我们把更加专注于核心业务创新,那么前端的工具主要解决的事情是项目管理工具、  插件依赖管理、编译管理、程序的入口和出口规范、 压缩或者去掉不需要的代码注释、还有代码风格检查以及自定义插件扩展功能这些等.

* JSLint/JSHint/ESLint/JSCS: 一个插件化并且可配置的 JavaScript 语法规则和代码风格的检查工具,现在ESLint是主流.

* Grunt/Webpack/Gulp: 一个集模块打包+控制管理+编译处理+过程控制于一体的前端自动化开发构建工具,grunt稍过气,现在比较流行的是grunt和webpack,这两个侧重点不同,可以结合使用.

* YUICompressor/JSMin/Tpacker-minifier:  是一些用来压缩 JS 和 CSS 文件的工具.

* JSDoc3/Doxx/YUIDoc: 前端代码用来生成HTML版API文档的一系列工具集 ,如果你的JavaScript文档符合规定的注释规范,并且命令书写正确,那么正确的JavaScript文档就会出现在你眼前.


# 标准规范 
&nbsp;&nbsp;&nbsp;&nbsp; 每个编程语言都有一套规则,只有大家都遵守约定好的规则,才能更好的发展,JavaScript/Html/Css都有本身的标准,里面涉及到代码规范、命名规范、标签规范、运行规范等等,那么就简单下介绍下前端涉及到的一些规范标准.

* HTML5/CSS3: 最新的html和css规范.

* ECMAScriupt*: JavaScript的一个标准规范.

* CommonJS/NodeJS: 非浏览器的应用标准,这些应用可以运行在不同的JavaScript解释器和不同的主机环境中,类似Ruby和Java标 准库.

* HTTP1.0/HTTP1.1/HTTP1.2: 互联网上应用最为广泛的一种网络协议.

* w3c/Dom/Bom/Xhtml/XML/JSON/JSONP: 浏览器对象规范和模型以及轻量级的数据交换格式JSON规范.


# 前端框架 
&nbsp;&nbsp;&nbsp;&nbsp;  前端框架实在是太多了,去看社区,基本每个月就会有一个新的出来,难道是写前端框架太容易了?而且大点的的公司都有不止一套自己的前端库或者组件库,这其实也反映了一个问题,就是各自为政,因为每个框架有很多相似性,某个稍微有点底子的前端程序员一看不惯别人的东西,就自己造,然后就出来一个框架了,其实这个冲劲是好的,但是如果造成了太多了,对入门前端的同学来说,不见得是好事啊,不知道学那个,一直学不完啊,我就大致写几个吧.

* Jquery/YUI/EXtjs: 老牌的JS库,在当前也是非常流行,基本影响着后来诸多框架的发展形态.

* Redux/MobX: React 和 MobX 是一对强力组合,React 通过提供机制把应用状态转换为可渲染组件树并对其进行渲染.而MobX提供机制来存储和更新应用状态供 React 使用.

* ramda: 函数式编程风格而设计.

* WeUI: 微信官方团队为微信Web开发打造的一个样式库,类似于 Bootstrap.

* ColorUI: 是小程序组件库.

* Omi: 腾讯出品的跨终端框架.

* Taro: 京东出品的跨平台框架.

* DoKit: 全称为DoraemonKit, 是滴滴推出一个跨平台的功能集合面板.

* MooTools: 面向对象的JS框架.

* prototype: 是应用最为广泛的Ajax开发框架.

* Jasmine: 是一个Javascript的BDD（Behavior-Driven Development）测试框架,不依赖任何其他框架.

* QUnit: jQuery 单元测试框架.


# 性能调优 
&nbsp;&nbsp;&nbsp;&nbsp; 性能在前端领域特别重要,因为如果页面元素渲染慢或者JS执行多久太复杂或一次加载了太多的JS文件,或者图片加载太大,这些都会影响客户体验,因此需要一些工具是真实的验证下页面的各个性能指标,看是否符合前期制定的规则和策略,更好的反应真实的用户,如下这些就是常用的性能工具.

* WebPageTest: 一款前端性能(析工).

* ShowSlow/YSlow: 用来收集页面性能(试工).

* PageSpeed Insights: 是一款可以分析网页速度瓶颈,让网页设计人员开发出速度更快的网站的谷歌浏(器插).

* HttpWatch: 是强大的网页数据分析工具.集成在Internet Explore(工具).

* dynaTrace Ajax Edition: 是一个强大的底层追踪、前端性能(析工).

* OneAPM: 为企业和开发者提供APM解决方案的服务商(商用).
